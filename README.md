# AI学习教育系统 - 小白项目说明文档

## 1️⃣ 项目是做什么的

这是一个**AI学习教育系统**，简单来说就是一个专门用于学习的网站，它可以根据不同用户角色（学生、教师、管理员）提供不同的功能。

**举个例子**：
- 如果你是学生，登录后可以看到「AI助教」「学习进度」「学习工具」等功能
- 如果你是教师，登录后可以看到「数据分析」「作业管理」「课程管理」等功能
- 如果你是管理员，登录后可以管理整个系统的用户和权限

## 2️⃣ 项目使用了哪些技术

### 核心技术

| 技术 | 是什么 | 做什么用 |
|------|--------|----------|
| **Vue 3** | 一个流行的前端框架 | 用于构建网站的界面和交互功能 |
| **TypeScript** | JavaScript的加强版 | 让代码更安全，减少错误 |
| **Vite** | 一个现代的前端构建工具 | 用于快速搭建和运行项目 |
| **Naive UI** | 一个UI组件库 | 提供现成的按钮、表单、菜单等界面元素 |
| **UnoCSS** | 一个CSS框架 | 用于美化网站的样式 |
| **Pinia** | 状态管理工具 | 用于管理网站的数据和状态 |

### 辅助技术

| 技术 | 作用 |
|------|------|
| **pnpm** | 用于安装和管理项目的依赖包 |
| **ECharts** | 用于制作各种图表，比如学习进度图、数据分析图 |
| **Vue Router** | 用于管理网站的页面跳转 |
| **vue-i18n** | 用于实现网站的多语言支持（中文/英文） |

## 3️⃣ 项目目录结构说明

项目的目录结构就像一个文件夹树，每个文件夹都有不同的用途：

```
fgwh/          # 项目根目录
├── public/     # 存放静态资源（如网站图标）
├── src/        # 项目的主要代码目录
│   ├── assets/      # 图片、SVG图标等资源
│   ├── components/  # 可复用的界面组件
│   ├── layouts/     # 网站的布局模板
│   ├── locales/     # 多语言配置文件
│   ├── router/      # 页面路由配置
│   ├── service/     # 与后端通信的服务
│   ├── store/       # 数据状态管理
│   ├── views/       # 网站的各个页面
│   ├── App.vue      # 应用的根组件
│   └── main.ts      # 应用的入口文件
├── package.json      # 项目配置文件
└── README.md         # 项目说明文档
```

### 关键目录详解

- **src/views/**：存放网站的所有页面，比如学生页、教师页、AI助教页等
- **src/components/**：存放可复用的界面元素，比如按钮、菜单、表格等
- **src/router/**：管理页面之间的跳转，比如从登录页跳转到首页
- **src/layouts/**：定义网站的整体布局，比如顶部导航栏、左侧菜单等
- **src/service/**：处理与后端服务器的通信，获取和发送数据

## 4️⃣ 项目运行流程

当你打开这个网站时，发生了以下过程：

1. **加载入口文件**：浏览器首先加载 `src/main.ts` 文件，这是整个应用的入口
2. **初始化应用**：创建Vue实例，设置路由、状态管理等
3. **渲染根组件**：将 `App.vue` 渲染到页面上
4. **加载布局**：根据当前页面选择合适的布局模板
5. **渲染页面**：加载并渲染当前URL对应的页面内容
6. **获取数据**：如果页面需要数据，会通过 `service` 模块从后端获取
7. **用户交互**：用户可以与页面交互，触发各种功能

## 5️⃣ 如何在本地部署并运行

### 准备工作

在开始之前，你需要安装以下软件：

1. **Node.js**：用于运行JavaScript代码，下载地址：https://nodejs.org/（选择LTS版本）
2. **VS Code**：用于查看和编辑代码，下载地址：https://code.visualstudio.com/

### 详细步骤

#### 第一步：安装依赖

打开命令行工具（Windows系统使用PowerShell或CMD），执行以下命令：

```cmd
# 进入项目目录
cd d:\fgwh

# 安装项目依赖
pnpm install
```

**为什么要运行这个命令？**：项目需要很多依赖包才能运行，这个命令会自动下载并安装所有必需的包。

#### 第二步：启动开发服务器

安装完成后，执行以下命令：

```cmd
# 启动开发服务器
pnpm dev
```

**为什么要运行这个命令？**：这个命令会启动一个本地服务器，让你可以在浏览器中查看和测试网站。

#### 第三步：访问网站

命令执行成功后，你会看到类似这样的输出：

```
  VITE v7.0.0  ready in 500 ms
  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ➜  press h + enter to show help
```

复制 `http://localhost:5173/` 到浏览器中访问，就可以看到网站了！

### 常见错误及解决方法

1. **错误**：`pnpm command not found`
   **解决方法**：先安装pnpm：`npm install -g pnpm`

2. **错误**：安装依赖时网络超时
   **解决方法**：检查网络连接，或使用国内镜像：`pnpm config set registry https://registry.npmmirror.com`

3. **错误**：启动服务器时端口被占用
   **解决方法**：使用其他端口：`pnpm dev --port 5174`

## 6️⃣ 项目中最重要的3～5个文件讲解

### 1. `src/main.ts` - 应用的入口文件

**作用**：这是项目的「大脑」，负责初始化整个应用。

**主要功能**：
- 导入并配置Vue实例
- 设置路由和状态管理
- 注册全局组件和插件
- 将应用挂载到浏览器中

### 2. `src/App.vue` - 应用的根组件

**作用**：这是项目的「骨架」，所有页面都会嵌套在这个组件中。

**主要功能**：
- 定义应用的基本结构
- 处理全局的布局和样式
- 包含网站的主要容器

### 3. `src/router/index.ts` - 路由配置文件

**作用**：这是项目的「导航员」，负责管理页面之间的跳转。

**主要功能**：
- 定义所有页面的URL路径
- 配置页面的访问权限
- 处理页面跳转逻辑

### 4. `src/store/index.ts` - 状态管理文件

**作用**：这是项目的「仓库」，负责管理网站的数据。

**主要功能**：
- 存储用户信息、登录状态等
- 管理全局的数据共享
- 处理数据的更新和同步

### 5. `src/service/request/index.ts` - 网络请求文件

**作用**：这是项目的「通信兵」，负责与后端服务器通信。

**主要功能**：
- 发送请求获取数据
- 处理响应数据
- 统一管理API接口

## 7️⃣ 总结：这个项目适合什么水平的人学习

这个项目适合**有一定前端基础**的人学习，特别是已经掌握了Vue 3和TypeScript的开发者。

**为什么这么说？**：
- 项目使用了现代化的前端技术栈，适合想学习最新前端技术的人
- 项目结构清晰，有完整的角色权限管理，适合学习企业级应用开发
- 项目包含了AI助教、数据分析等实用功能，适合学习实际项目的开发流程

**对于零基础的新手**：建议先学习HTML、CSS、JavaScript的基础知识，再学习Vue 3和TypeScript，然后再来学习这个项目。
